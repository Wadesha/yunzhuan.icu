可用性评估

Usability Evaluation

课程介绍 Course Introduction

学分：3 | 先修课：用户研究方法 | 学期：第5学期

可用性评估课程系统介绍评估交互系统可用性的理论、指标与方法。课程涵盖专家评估法（启发式评估、认知走查）、用户测试法（实验室测试、远程测试、眼神追踪）、主观量表（SUS、NASA-TLX）与数据分析技术。学生将学习根据评估目标与资源约束选择合适方法，科学度量效率、有效性、满意度等可用性维度，并提出可执行的改进建议。

Usability Evaluation systematically introduces theories, metrics, and methods for assessing system usability. Topics include expert evaluation methods such as heuristic evaluation and cognitive walkthrough, user testing including lab, remote, and eye-tracking studies, subjective scales such as SUS and NASA-TLX, and data analysis techniques. Students learn to choose appropriate methods and measure usability dimensions scientifically.

大作业 Final Project

作业标题：数字产品可用性评估与改进方案设计

选择一款数字产品，综合运用启发式评估、用户测试与主观量表完成可用性评估，并提出改进建议。提交评估方案、数据分析与改进设计报告。

Select a digital product, conduct a usability evaluation using heuristic evaluation, user testing, and subjective scales, and propose improvements. Deliver an evaluation plan, data analysis, and an improvement design report.

实施步骤 Implementation Steps

📋 示例：对一款常用App做系统的可用性评估，比如某外卖App的下单流程。你需要招募5-8名用户做有声思维测试，记录任务完成时间和错误次数，用SUS量表打分，找出3-5个关键问题并提出改进建议。
步骤 1
评估方案设计
本步骤进行用户研究与需求探索，深入理解可用性评估的目标用户与使用场景。用户研究是UX设计的起点，通过定性与定量研究方法获取用户洞察。进行用户访谈、问卷调查、焦点小组、可用性测试等研究活动。建立用户画像（Persona）与用户旅程图，明确用户的目标、痛点与期望。用户研究为后续设计决策提供数据支撑与用户视角。

• 用户研究方法选择：定性研究（用户访谈、上下文访谈、焦点小组）与定量研究（问卷、数据分析）结合
• 用户画像建模：建立典型用户角色（Persona），包含人口统计学特征、目标、行为模式、痛点需求
• 场景与旅程分析：定义使用场景，绘制用户旅程图（User Journey Map），识别关键触点与痛点
产出：用户研究报告（含研究方法、用户画像、场景分析、痛点清单）｜ 质量标准：研究深入，画像真实，洞察有价值
步骤 2
专家评估
本步骤进行可用性评估的信息架构与交互设计。信息架构设计组织与结构化产品内容与功能，确保用户能高效找到所需信息。进行站点地图、导航系统、分类体系设计。交互设计定义用户与产品的交互方式，设计页面流程与交互反馈。运用交互设计原则与模式，创建线框图（Wireframe）与交互原型。

• 信息架构设计：站点地图（Sitemap）、导航设计、分类体系、标签设计、搜索策略
• 交互流程设计：任务流设计、页面流程图、状态转换设计、微交互设计、反馈机制
• 线框与原型：低保真线框图（Balsamiq/Sketch）、交互原型（Axure/Figma/Prototype）、设计规范
产出：交互设计方案（含信息架构、流程图、线框图、可交互原型）｜ 质量标准：架构清晰，流程顺畅，原型可用
步骤 3
用户测试
本步骤进行可用性评估的视觉设计与品牌呈现。视觉设计将交互框架转化为美观、可用的界面，建立产品的视觉语言与品牌调性。制定设计系统与设计规范，包括色彩体系、字体规范、图标风格、组件库。设计高保真界面原型，确保视觉一致性与品牌识别度。运用视觉设计原则（对比、对齐、层级、留白）提升界面品质。

• 设计系统建立：色彩体系（主色/辅助色/中性色）、字体规范（字号/字重/行高）、栅格系统、组件库
• 视觉界面设计：高保真设计稿（Figma/Sketch）、图标设计、插画设计、动效设计、响应式设计
• 设计规范与交付：UI Kit、设计手册、切图标注、设计交付物清单、设计评审
产出：视觉设计方案（含设计系统、高保真设计稿、设计规范、UI Kit）｜ 质量标准：视觉美观，规范统一，符合品牌调性
步骤 4
量表与数据分析
本步骤进行可用性测试与可用性评估的设计评估。可用性测试是检验设计质量的关键环节，通过真实用户的使用反馈发现设计问题。招募目标用户，设计测试任务与场景，进行实验室测试或远程测试。收集并分析定量数据（任务完成率、时间、错误率）与定性数据（用户反馈、满意度评分）。根据测试结果迭代优化设计方案。

• 测试方案设计：确定测试目标与方法（形成性/总结性），设计测试任务与场景，编写测试脚本
• 用户测试执行：招募目标用户，进行一对一测试（有声思维法），记录行为与反馈，使用Morae/Observer
• 数据分析与报告：定量分析（任务完成率、SUS评分、NPS）、定性分析（亲和图、痛点排序）、改进建议
产出：可用性测试报告（含测试方案、测试记录、数据分析、改进建议）｜ 质量标准：测试科学，数据充分，改进明确
步骤 5
改进设计与报告
本步骤进行设计交付与项目总结，完成可用性评估课程大作业的最终产出。设计交付是将设计成果移交给开发团队的过程，需要提供完整、规范的设计交付物。进行设计走查与开发协作，确保设计还原度。总结整个设计过程，反思设计决策与成果。整理设计资产，建立可复用的设计组件库。良好的交付与总结是专业设计师的必备素养。

• 设计交付物整理：设计规范、切图资源、标注文档、设计源文件、交互原型、设计说明
• 开发协作与走查：设计评审会、开发走查、还原度检查、设计变更管理、组件库维护
• 项目总结：设计复盘报告、成果展示、经验教训、设计资产归档、未来迭代建议
产出：设计交付物（含设计稿、规范文档、切图资源、总结报告）｜ 质量标准：交付规范，协作顺畅，总结深入

Steps

Step 1
Evaluation Plan Design
This step conducts user research and requirements exploration, deeply understanding target users and usage scenarios of Usability Evaluation. User research is the starting point of UX design, obtaining user insights through qualitative and quantitative research methods. Conduct user interviews, questionnaires, focus groups, usability testing and other research activities. Create Personas and user journey maps to clarify user goals, pain points and expectations. User research provides data support and user perspective for subsequent design decisions.

• User research method selection: combine qualitative research (user interviews, contextual inquiry, focus groups) and quantitative research (surveys, data analysis)
• Persona modeling: create typical user roles (Personas), including demographic characteristics, goals, behavior patterns, pain points and needs
• Scenario and journey analysis: define usage scenarios, draw User Journey Maps, identify key touchpoints and pain points
Deliverable: User research report (research methods, personas, scenario analysis, pain point list) | Quality standard: In-depth research, realistic personas, valuable insights
Step 2
Expert Evaluation
This step conducts information architecture and interaction design for Usability Evaluation. Information architecture design organizes and structures product content and functions, ensuring users can efficiently find needed information. Conduct sitemap, navigation system, classification system design. Interaction design defines interaction methods between users and products, designs page flows and interaction feedback. Apply interaction design principles and patterns, create wireframes and interactive prototypes.

• Information architecture design: Sitemap, navigation design, classification system, tag design, search strategy
• Interaction flow design: task flow design, page flowcharts, state transition design, micro-interaction design, feedback mechanism
• Wireframing and prototyping: low-fidelity wireframes (Balsamiq/Sketch), interactive prototypes (Axure/Figma/Prototype), design specifications
Deliverable: Interaction design solution (information architecture, flowcharts, wireframes, interactive prototype) | Quality standard: Clear architecture, smooth flow, usable prototype
Step 3
User Testing
This step conducts visual design and brand presentation for Usability Evaluation. Visual design transforms interaction framework into beautiful, usable interfaces, establishing product visual language and brand tone. Develop design system and design specifications, including color system, typography, icon style, component library. Design high-fidelity interface prototypes, ensuring visual consistency and brand recognition. Apply visual design principles (contrast, alignment, hierarchy, whitespace) to improve interface quality.

• Design system establishment: color system (primary/secondary/neutral), typography (font size/weight/line height), grid system, component library
• Visual interface design: high-fidelity design drafts (Figma/Sketch), icon design, illustration design, motion design, responsive design
• Design specification and delivery: UI Kit, design manual, cut files and annotations, design deliverable list, design review
Deliverable: Visual design solution (design system, high-fidelity designs, design specifications, UI Kit) | Quality standard: Beautiful visuals, unified specifications, matches brand tone
Step 4
Scales & Data Analysis
This step conducts usability testing and design evaluation for Usability Evaluation. Usability testing is a keystep of verifying design quality, discovering design problems through real user feedback. Recruit target users, design test tasks and scenarios, conduct lab testing or remote testing. Collect and analyze quantitative data (task completion rate, time, error rate) and qualitative data (user feedback, satisfaction scores). Iteratively optimize design solutions based on test results.

• Test plan design: determine test objectives and methods (formative/summative), design test tasks and scenarios, write test script
• User test execution: recruit target users, conduct one-on-one testing (think-aloud method), record behavior and feedback, use Morae/Observer
• Data analysis and reporting: quantitative analysis (task completion rate, SUS score, NPS), qualitative analysis (affinity diagram, pain point ranking), improvement suggestions
Deliverable: Usability test report (test plan, test records, data analysis, improvement suggestions) | Quality standard: Scientific testing, sufficient data, clear improvements
Step 5
Improvement Design & Report
This step conducts design delivery and project summary, completing final output of the Usability Evaluation course project. Design delivery is the process of transferring designresults to development team, requiring complete and standardized design deliverables. Conduct design walkthrough and development collaboration to ensure design fidelity. Summarize the entire design process, reflect on design decisions and outcomes. Organize design assets, establish reusable design component library. Good delivery and summary skills are essential qualities for professional designers.

• Design deliverable organization: design specifications, cut resources, annotation documents, design source files, interactive prototype, design description
• Development collaboration and walkthrough: design review meeting, development walkthrough, fidelity check, design change management, component library maintenance
• Project summary: design retrospective report,resultsdisplay, lessons learned, design asset archiving, future iteration suggestions
Deliverable: Design deliverables (design drafts, specification documents, cut resources, summary report) | Quality standard: Standardized delivery, smooth collaboration, in-depth summary
